# 应用指南

这份指南介绍 OxideTerm 桌面应用本身。日常终端、SSH、文件、转发、AI、插件、同步和设置工作都从桌面应用开始。`oxideterm` CLI 是单独的伴侣工具，用于自动化、诊断、CI、迁移和恢复。

## 首次启动

从系统启动器打开 OxideTerm。第一次使用时，建议先打开一个本地终端标签页，再添加远端主机。本地终端可以先确认当前机器上的渲染、键盘输入、shell 启动、字体和主题设置都正常。

推荐的首次流程：

1. 打开一个本地终端。
2. 输入简单命令，例如 `pwd` 或 `echo ok`。
3. 打开设置，检查终端字体、主题、shell 和 AI 设置。
4. 添加一个保存的 SSH 连接。
5. 连接主机，并验证终端输入、文件浏览和连接状态。

## 应用布局

主窗口围绕带标签页的工作区组织，左侧是活动栏和上下文侧边面板。

主要区域包括：

- 会话：保存的连接、活跃 SSH 节点和终端会话。
- 连接池：可复用的连接运行时状态。
- 连接监控：健康状态和重连状态。
- 连接矩阵：更完整的连接概览。
- 文件管理器与 SFTP：浏览、预览、上传、下载和编辑远端文件。
- 插件：安装、启用、禁用和配置插件。
- 云同步：同步应用状态并查看同步状态。
- 通知：查看应用事件和警告。
- 设置：配置应用行为。

标签页是主要工作面。一个标签页可以是本地终端、SSH 终端、SFTP、IDE 工作区、设置页、文件管理器、插件管理器、连接监控或其他应用页面。

## 标签页与面板

用标签页分隔不同任务。同一个任务需要多个 shell 并排工作时，在终端标签页里使用分屏面板。

终端状态属于具体面板。关闭面板会关闭可见终端，但不等于删除保存的连接配置。远端工作中，连接运行时状态和可见终端标签页是分开管理的。

典型终端流程：

1. 打开或选择一个保存连接。
2. 启动 SSH 终端。
3. 需要时拆分终端面板。
4. 通过命令标记和 shell 历史跟踪输出。
5. 长时间任务或不稳定连接，保持连接监控可见。

## 保存连接

经常复用的主机应该保存为连接。保存连接可以包含：

- 名称、host、port 和 username。
- 用于导航的 group、color 和 tags。
- 认证方式，例如 SSH agent、密钥、密码或默认 SSH 行为。
- 可选的连接行为，例如连接后命令。
- 在配置支持时使用代理或跳板机。

分组和标签只用于导航。不要把密码、令牌或凭据写进名称、分组、标签或备注。

## 连接主机

在“会话”区域选择保存连接并打开。连接成功后，应用会为 SSH 节点创建运行时状态，并打开终端。

如果主机断开，通过连接池或连接监控判断运行时是正在重连、已失效还是不可用。应从应用状态里重连，不要为了重连而重新创建保存连接。

## SFTP 与远端文件

使用 SFTP 或文件管理器处理远端文件：

- 浏览远端目录。
- 下载或编辑前先预览文件。
- 上传和下载文件。
- 启动大文件传输并跟踪进度。
- 连接不稳定时，重连后再重试传输。

覆盖重要远端文件前，先确认路径并保留备份。远端写入就是目标系统上的真实写入。

## IDE 工作区

需要项目式远端文件工作流时，使用 IDE 页面。为已连接节点打开 IDE 工作区，选择远端目录，然后在编辑器标签页中处理项目文件。

IDE 工作区状态和普通终端标签页不同。当前编辑器标签页、项目根目录、未保存缓冲区和打开文件都属于 IDE 页面。连接中断后，应先重连，再依赖保存或项目搜索结果。

## 端口转发

使用转发界面管理本地、远端和动态转发：

- 本地转发：本地端口连接到远端目标。
- 远端转发：远端端口反连到本地目标。
- 动态转发：创建 SOCKS 风格隧道。

只有当转发应随所属连接自动启动时才启用自动启动。测试新转发时建议保持连接监控可见。

## AI 侧边栏

AI 侧边栏会结合当前应用上下文工作。启用工具调用后，它可以检查目标、使用终端和文件工具，并对当前工作区状态做总结或操作。

推荐 AI 使用流程：

1. 先打开相关终端、连接、SFTP、IDE 或设置页面。
2. 只在需要时包含上下文。
3. 让 AI 先列出或选择明确目标，再运行命令。
4. 对写入、交互式或破坏性操作，认真检查审批提示。
5. 接受后续修改前，先看工具结果。

不要把凭据粘贴进 AI 提示词。API key 或凭据应使用应用里的供应商密钥和凭据存储页面。

## 设置

交互式配置优先使用设置页面：

- 通用应用行为。
- 外观和主题。
- 终端 renderer、shell、字体、编码和本地终端行为。
- SSH、重连、SFTP 和 IDE 行为。
- AI 供应商、模型选择、记忆、工具调用和知识库设置。
- 快捷键、帮助和更新信息。

探索性或视觉类修改优先用桌面 UI。只有需要脚本化或跨环境重复执行时才使用 CLI。

## 插件

使用插件管理器安装、启用、禁用、更新和配置插件。启用非本人编写的插件前，先查看插件权限和设置。

插件设置和插件凭据应通过专门的应用页面管理。不要把凭据写进插件名称、标签或普通文本字段，除非该字段明确是凭据字段。

## 云同步与备份

需要查看同步状态、手动同步或处理可见同步问题时，使用桌面应用里的云同步页面。执行高影响导入、恢复或同步变更前，保持备份可用。

需要支持包或自动化恢复计划时，在桌面应用中确认问题后再使用 CLI 伴侣工具。

## CLI 伴侣工具

CLI 伴侣工具适合无界面、可重复或诊断类工作：

- `doctor` 和支持报告。
- 脚本化设置变更。
- 无界面连接校验。
- 备份、恢复和云同步自动化。
- CI 中检查导出的配置。

日常使用先从桌面应用开始。CLI 应辅助桌面工作流，而不是替代桌面应用。
