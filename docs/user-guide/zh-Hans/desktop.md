# 桌面工作流

## 首次启动

打开 OxideTerm 后，先看左侧活动栏。主要区域包括会话、连接池、连接监控、连接矩阵、插件、云同步、文件管理器、通知和设置。

如果当前没有会话，建议先创建一个本地 shell 标签页。这可以先确认终端渲染、shell integration、输入处理和主题设置都正常，再去添加远端主机。

## 终端面板

终端标签页可以用于本地 shell 和 SSH 会话。一个任务需要多个 shell 时，使用分屏面板。命令标记、shell integration 和终端历史属于具体面板；关闭面板不应该等同于断开保存的 SSH 主机。

对于长时间运行的任务，保留连接池或连接监控里的状态可见。重连行为跟连接/runtime 状态相关，不只取决于当前有没有可见终端标签页。

## 保存连接

经常复用的主机应该保存为连接。连接配置包含 host、user、port、group、color、tags、认证方式和可选的连接后命令。可以的话优先使用 SSH agent 或密钥认证。

分组用于导航和批量整理，不要把 secret 或环境密码编码进分组名。

## 文件管理器与 SFTP

文件管理器用于远端浏览、上传、下载、预览和基础文件操作。远端编辑就是实际远端写入：关键文件先备份，覆盖前确认路径。

连接不稳定时，先暂停大文件传输，重连后再重试。保存连接状态和传输状态是分开的；传输失败不应该要求删除连接。

## 设置

设置按功能区组织。外观、终端行为、AI provider、云同步、portable runtime、帮助与关于等交互式配置优先用桌面 UI 修改。

脚本化或可重复的修改使用 CLI，并先加 `--dry-run`。CLI 和桌面应用读取同一套配置文件。

## 更新

在“设置 → 帮助与关于”里查看当前版本和更新通道。稳定版、beta 和 GPUI preview 使用独立更新通道，应该选择与当前安装包匹配的通道。
