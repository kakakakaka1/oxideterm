# 排障

排障先从桌面应用开始。可见应用状态通常能告诉你问题来自保存配置、在线 SSH 节点、终端会话、主机工具、图形/VNC、SFTP、转发、同步、设置还是插件。

## 先检查应用页面

修改文件或运行修复命令前，先检查相关页面：

- 会话：确认保存连接存在，并且 host、user、port、group 和认证方式符合预期。
- 连接监控：确认节点是已连接、正在连接、已失效、正在重连还是不可用。
- 主机工具：确认资源快照是否新鲜，以及动作是否带有可见错误。
- 终端标签页：确认 shell 是否接受输入、命令是否仍在运行，以及是否有终端辅助提示处于活跃状态。
- 图形/VNC：确认所属节点在线，并且查看器已连接。
- SFTP 或文件管理器：重试目录读取或传输前，确认目标节点在线。
- 设置：检查最近是否修改过终端背景图片、提权凭据、SSH、AI、云同步、插件或更新设置。
- 通知：查看最近的警告和错误。

如果某个连接或页面已失效，先从应用里重连，再考虑改配置。

## 常见恢复步骤

设置问题：重新打开设置，检查最近修改的部分。如果应用报告设置无效，先回退最小的可见变更。

连接问题：编辑原保存连接，然后从“会话”重试。确认原配置错误前，不要创建重复连接。

SFTP 或转发问题：先在连接监控里检查所属 SSH 节点。节点在线后再重试。

主机工具问题：先刷新工具页面。如果采样器或动作仍失败，重连所属节点，并重试最小动作。不要把主机工具当作隐藏清理工具或递归磁盘扫描工具。

图形/VNC 问题：先检查所属节点，重连查看器；如果后端进程已停止，再重启图形会话。查看器状态与终端输出、保存连接数据分离。

终端背景问题：重新打开设置，确认背景图片已对当前标签页类型启用。Native 当前把背景视为单个已选图片槽；添加新图片会替换当前选择。

全屏 TUI 退出后留下旧块：先尝试 `clear` 或重新打开终端面板。如果 `yazi` 这类命令能稳定复现，应按终端图形/image-placement 状态问题处理，并在问题报告里写明命令名。

X/Y/ZMODEM 传输问题：意外弹出的提示先取消。用明确的传输命令重试，例如 `rz`、`sz`、`rx` 或 `rb`，然后在应用提示中选择本地路径。普通命令输出不应在协议上下文不明确时被当作传输。

提权凭据问题：检查专用设置页面和活跃终端面板。调试时不要把 sudo/su 密码粘贴进日志、AI 提示词、支持包、快捷命令或连接备注。

云同步问题：打开云同步页面，检查状态并查看冲突，再选择同步方向。

串口终端问题先从设备和权限边界排查：

- 未列出串口：手动输入 `/dev/cu.*`、`/dev/ttyUSB*`、`/dev/ttyACM*` 或 `COMx`，并确认系统能看到该设备。
- 权限被拒绝：Linux 检查 `dialout`、`uucp` 或发行版对应串口组，修改组后退出登录再进入。macOS 检查系统权限和 USB 串口驱动。
- 设备忙：关闭可能占用该端口的其他终端程序、调试器、烧录工具或 OxideTerm 标签页。
- 设备拔出：关闭当前串口终端，重新插入设备后再打开。如果系统分配了新路径，更新串口配置。

## 先备份

执行恢复、导入、同步应用或手动修复文件前，先从应用里创建或确认备份。检查恢复计划，只应用能解决问题的最小部分。

## CLI 伴侣工具诊断

需要只读诊断、CI 检查或支持包时使用 CLI 伴侣工具：

```sh
oxideterm paths --json
oxideterm diagnose --json
oxideterm doctor --strict
oxideterm report --json
```

`doctor --strict` 会把警告当作失败，适合 CI 或迁移脚本。

针对性检查：

```sh
oxideterm settings validate --strict
oxideterm connections validate --strict
oxideterm cloud-sync status --json
```

## 问题报告

提交问题报告时优先附加脱敏支持包，而不是原始配置文件：

```sh
oxideterm report --bundle ./oxideterm-report.json --json
```

分享前先检查支持包。必要时移除私人主机名、用户名、路径或项目名。
