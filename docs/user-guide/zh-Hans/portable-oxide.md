# 便携 `.oxide` 包

`.oxide` 是加密的便携导入导出包，用于在机器或配置档之间迁移 OxideTerm 数据。它可以包含连接、转发、应用设置、快捷命令、插件设置、托管 SSH 密钥，以及可选的便携凭据。

普通导入导出优先使用桌面应用，这样可以在应用里预览变更后再执行。

## 导入前预览

打开便携导入页面，选择 `.oxide` 文件，先预览内容再导入。重点检查：

- 包里包含哪些连接、转发、设置、快捷命令和插件设置。
- 是否包含托管 SSH 密钥，以及是否会恢复到 OxideTerm。
- 是否包含便携凭据。
- 哪些记录会和当前配置档冲突。
- 将使用哪种冲突策略。

预览结果不符合预期前，不要导入便携包。

## 导入策略

选择冲突策略：

- `skip`：保留已有本地记录。
- `rename`：冲突记录用新名字导入。
- `replace`：替换本地记录。
- `merge`：合并兼容记录。

使用能满足任务的最小策略。例如，只是查看另一台机器的包时优先使用 `skip` 或 `rename`；只有明确要让包覆盖本地记录时才用 `replace`。

## 导出

在导出页面选择要包含的内容。添加清晰描述，方便目标机器之后识别这个便携包。

凭据材料是显式选项：

- 保存的服务器密码默认不包含。
- 外部私钥文件只有启用嵌入时才会复制进包。
- 已保存的密钥或证书 passphrase 有单独的可见选项。
- OxideTerm 托管 SSH 密钥可以包含进包，使托管密钥连接在导入后恢复到托管密钥库。
- 托管密钥 passphrase 默认不包含，除非导出时显式开启。
- 便携凭据用于便携迁移等自包含配置档迁移场景。

只有目标机器需要加密凭据材料时，才包含便携凭据。只有明确需要把私钥文件或托管密钥放进加密包时才开启，并确保便携包密码足够强。

## 托管 SSH 密钥

托管密钥是 OxideTerm 自己管理的凭据。连接配置只保存托管密钥引用，私钥材料保存在本地钥匙串或便携密钥库中。

当包里包含托管密钥时，导入预览会允许恢复到 OxideTerm。若关闭恢复，OxideTerm 可以在可用时使用内嵌 fallback 密钥文件；否则受影响连接导入后需要修复认证配置。

重复托管密钥按 fingerprint 匹配，应复用已有密钥，而不是再创建一份副本。

## Cloud Sync 边界

Cloud Sync 可以上传加密 `.oxide` 快照，但后台同步或插件驱动同步不会静默包含托管 SSH 密钥。需要在机器之间迁移完整凭据材料时，使用手动导出或便携迁移流程。

## 便携运行时

便携运行时密钥库用于保护导入后的便携凭据。通过应用里的便携运行时或凭据存储页面设置它。如果密钥库处于锁定状态，先解锁再依赖导入的便携凭据。

只有明确要删除本地便携密钥库时，才重置便携运行时。

## CLI 伴侣工具

自动化、CI 校验或脚本化迁移使用 CLI 伴侣工具：

```sh
oxideterm oxide validate ./profile.oxide
oxideterm oxide preview-import ./profile.oxide --password-stdin --json
oxideterm oxide diff ./profile.oxide --strategy merge --password-env OXIDE_PASSWORD
oxideterm portable status --json
```

便携包密码优先通过标准输入或环境变量提供。不要把密码直接写进 shell 历史。
