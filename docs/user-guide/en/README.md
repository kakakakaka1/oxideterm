# OxideTerm User Guide

OxideTerm Native is a Rust/GPUI SSH workspace with terminal, SFTP, forwarding, Host Tools, graphics/VNC sessions, settings, cloud sync, plugins, AI assistance, and a standalone CLI companion. The desktop app is the product surface; the CLI is an automation and diagnostics companion.

## Guides

- [Getting Started](./getting-started.md): first launch, local terminal check, saved connection setup, and configuration locations.
- [Application Guide](./app.md): app layout, tabs, sessions, SFTP, IDE, forwarding, Host Tools, graphics/VNC, AI, settings, plugins, and cloud sync.
- [Architecture](./architecture.md): how the desktop app is organized, including nodes, terminals, Host Tools, modem transfers, graphics/VNC, SFTP, IDE, AI, plugins, sync, security, and CLI boundaries.
- [Desktop Workflows](./desktop.md): practical workflows for terminal panes, connections, Host Tools, graphics/VNC, settings, and update checks.
- [CLI Companion](./cli.md): common headless workflows for diagnostics, settings, connections, backups, cloud sync, and automation.
- [Connections and Forwards](./connections-and-forwards.md): saved SSH profiles, live runtime state, Host Tools, graphics/VNC sessions, connection monitor, and port forwarding.
- [Cloud Sync and Backups](./cloud-sync-and-backups.md): sync status, manual sync, conflict review, backups, restore plans, and support bundles.
- [Portable `.oxide` Bundles](./portable-oxide.md): encrypted import/export for moving profiles between machines.
- [Plugins and Secrets](./plugins-and-secrets.md): plugin manager workflows, plugin settings, and secret-safe automation.
- [Native Plugin Development](./plugin-development.md): manifest-only, process, and WASM plugin development for the native app.
- [Troubleshooting](./troubleshooting.md): app-first checks, terminal helpers, connection recovery, Host Tools, graphics/VNC, sync recovery, diagnostics, and bug reports.
